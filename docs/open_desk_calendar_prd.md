# 开源桌面日历天气屏 PRD

版本：v0.3  
状态：产品需求定稿草案  
项目暂名：Open Desk Calendar / 开桌历  
目标平台：Android 4.1+ 老设备、旧平板、安卓电纸书、闲置手机  
核心目标：把闲置 Android 设备改造成一个开源、简洁、常亮、可自启的桌面日历天气屏。

---

## 1. 背景与目标

现有闭源桌面日历天气类软件能够满足“旧手机变桌面日历”的基本需求，但存在以下问题：

1. 不开源，无法审计代码和接口行为。
2. 天气接口、Key、Host、更新逻辑不透明。
3. 获取方式依赖私域渠道，不利于长期维护。
4. 低版本 Android 支持不确定，老设备兼容风险较高。
5. 电纸书、低刷新、低功耗、自启、Home 桌面模式等能力不够体系化。
6. 界面虽然可用，但仍可以进一步优化为“一眼看懂”的桌面信息屏。

本项目目标是做一个开源替代实现，并在以下方面超过原软件：

1. 支持更低 Android 版本。
2. 主界面更简洁，一屏内完成时间、天气、日历、农历、宜忌、Wi-Fi 状态展示。
3. 内置开机自启和 Home Launcher 能力，适合作为专用桌面屏。
4. 天气源可插拔，默认支持免 Key 天气源，也允许用户配置自定义 Host/Key。
5. 日历、农历、节气、法定节假日、调休数据可审计、可更新。
6. 支持浅色、深色、黑白、电纸书模式。
7. 默认无账号、无广告、无强制公众号、无默认遥测。

## 2. 产品定位

一句话定位：

> 一个面向旧 Android 设备的开源桌面日历天气屏，强调大时钟、实时天气、三日预报、万年历、农历、宜忌、开机自启和 Home 桌面模式。

产品关键词：

- 旧手机再利用
- 大字号数字时钟
- 桌面日历
- 天气预报
- 农历 / 节气 / 宜忌
- 法定假日 / 调休 / 补班
- 开机自启
- Home Launcher
- 电纸书友好
- 开源可审计

## 4. 版本范围

### 4.1 MVP 版本目标

MVP 必须实现：

1. 大号数字时钟。
2. 日期、星期显示。
3. 当前天气。
4. 三天天气预报。
5. 月历。
6. 农历、节气、传统节日。
7. 法定节假日和调休补班标记。
8. 当天日期高亮背景。
9. Wi-Fi 状态显示。
10. 城市设置。
11. 开机自启。
12. Home Launcher 模式。
13. 设置页。
14. 浅色、深色、黑白主题。
15. 断网缓存。

### 4.2 P1 增强目标

1. 宜 / 忌信息。
2. 室内温湿度接入。
3. 电纸书低刷新模式。
4. 自定义天气接口 Host + Key。
5. 备用启动器选择。
6. Home Assistant / MQTT 接入。
7. 多城市轮播。
8. 空气质量、天气预警。
9. 夜间自动降亮度。
10. 假日数据在线更新。

### 4.3 暂不做

MVP 阶段不做：

1. 账号系统。
2. 广告系统。
3. 云端同步。
4. 强制定位权限。
5. 复杂日程管理。
6. 小组件系统。
7. 应用商店商业分发。
8. 复制闭源应用代码、图标、素材、专有接口。

## 5. 兼容性要求

### 5.1 Android 版本

最低目标：

```text
minSdkVersion = 16
最低支持 Android 4.1 / API 16
优先兼容 Android 4.1、4.2、4.4、5.0、6.0
```

实验目标：

```text
尝试兼容 Android 4.0.3 / API 15
是否正式支持取决于真机测试结果
```

不建议目标：

```text
Android 4.0.1 / 4.0.2 及更低版本暂不承诺
```

### 5.2 技术约束

为了支持老设备，MVP 不使用 Jetpack Compose，不强依赖最新版 AndroidX。

建议技术方向：

```text
语言：Java 优先，Kotlin 谨慎使用
UI：原生 View + 自定义 Canvas 绘制
配置：SharedPreferences
缓存：SQLiteOpenHelper
网络：HttpURLConnection 或低版本兼容 OkHttp
后台：BroadcastReceiver + AlarmManager
日历：NCalendar 低版本 / Fork 版 / 自研 CalendarView
```

## 6. 主界面设计目标

参考视觉方向：

> 界面简洁，一目了然。左侧看时间和天气，右侧看日历和宜忌。

主界面要遵循以下原则：

1. 时间最大，一眼可读。
2. 当前日期、星期、农历清楚可见。
3. 天气信息集中，不分散。
4. 未来三天天气在时间下方展示。
5. 日历占据右侧主要区域。
6. 当天日期必须有明显背景色或边框。
7. 法定假日显示“休”，调休补班显示“班”。
8. Wi-Fi 状态显示在右上角。
9. 横屏优先，竖屏适配。
10. 电纸书模式不使用大面积深色填充。

## 10. 核心功能需求

### 10.1 大时钟

| 编号 | 需求 | 优先级 | 验收标准 |
|---|---|---:|---|
| CLK-01 | 显示大号小时和分钟 | P0 | 1 米外可清楚看到小时和分钟 |
| CLK-02 | 显示日期和星期 | P0 | 日期格式可配置为中文短格式或完整格式 |
| CLK-03 | 支持显示/隐藏秒 | P0 | 隐藏秒后每分钟刷新一次 |

| CLK-05 | 跨日自动刷新日期、星期、农历 | P0 | 00:00 后主界面数据自动更新 |

### 10.2 天气

| 编号 | 需求 | 优先级 | 验收标准 |
|---|---|---:|---|
| WEA-01 | 显示城市/区县 | P0 | 主界面左上角或天气区显示当前城市 |
| WEA-02 | 显示当前天气现象 | P0 | 例如晴、多云、阴、小雨、中雨 |
| WEA-03 | 显示当前温度 | P0 | 单位为摄氏度 |
| WEA-04 | 显示湿度 | P0 | 当前湿度可见 |
| WEA-05 | 显示风向风力 | P0 | 例如东北风 1级 |
| WEA-06 | 显示三天天气预报 | P0 | 今天、明天、后天均显示天气和温度范围 |
| WEA-07 | 显示数据更新时间 | P0 | 例如“更新于 21:40” |
| WEA-08 | 断网显示缓存 | P0 | 断网不空白，显示上次数据 |
| WEA-09 | 自定义 Host + Key | P1 | 用户可配置第三方天气接口 |
| WEA-10 | 局域网天气代理 | P1 | 老设备 HTTPS 失败时可改走局域网代理 |

### 10.3 室内温湿度

| 编号 | 需求 | 优先级 | 验收标准 |

| IND-03 | 无传感器时自动隐藏 | P0 | 未配置时界面不留明显空白 |

## 11. 自动启动与 Home Launcher 能力

### 11.1 开机自启

| 编号 | 需求 | 优先级 | 验收标准 |
|---|---|---:|---|
| BOOT-01 | 支持开机自启 | P0 | 设备重启后自动进入主界面 |
| BOOT-02 | 设置页可开关 | P0 | 用户可主动开启或关闭 |
| BOOT-03 | 首次安装不强制自启 | P0 | 需要用户主动开启 |
| BOOT-04 | 自启失败可诊断 | P1 | 最近错误中显示自启失败原因 |

### 11.2 内置 AnyLauncher 类能力

本项目内置“专用桌面模式”，实现类似 AnyLauncher 的核心能力，但不复制其代码。

| 编号 | 需求 | 优先级 | 验收标准 |
|---|---|---:|---|
| HOME-01 | 应用可作为 Home/Launcher | P0 | 用户可在系统中设置为默认桌面 |
| HOME-02 | 按 Home 键返回本应用 | P0 | 设置默认桌面后，按 Home 回到日历屏 |
| HOME-03 | 支持备用启动器 | P1 | 可选择系统桌面或其他 App 作为备用入口 |
| HOME-04 | 防锁死机制 | P0 | 长按角落或连续点击可进入设置/备用启动器 |
| HOME-05 | 不静默修改默认桌面 | P0 | 必须由用户在系统界面确认默认 Home |
| HOME-06 | 退出需要确认 | P1 | 可设置是否二次确认退出 |

### 11.3 防锁死规则

必须实现：

1. 长按时间区域进入设置。
2. 长按右上角 Wi-Fi 区域进入系统/备用启动器入口。
3. 设置页提供“打开系统桌面选择器”或“打开备用启动器”。
4. 设置页提供“关闭 Home 模式提示”。
5. 不允许把用户困在本应用里无法返回系统。

## 16. NCalendar 使用要求

项目可以使用 `yannecer/NCalendar`，但必须满足低版本 Android 的兼容目标。

关键要求：

1. 不直接使用要求 Android 8.0+ 的新版 NCalendar 作为 MVP 依赖。
2. 优先评估 NCalendar 5.x / 4.x 或 Fork 低版本分支。
3. 如果库版本无法满足 Android 4.1+，则只借鉴其设计思路，使用自研 CalendarView。
4. 日历视觉必须自定义，不能直接套用默认 UI。
5. 当天日期必须有背景色、边框或反色块。
6. 休/班角标由本项目假日数据包控制。
7. 宜/忌数据源需要单独确认授权和准确性。

NCalendar 集成方向：

```text
优先使用：月视图、农历、节气、日期区间、自定义 CalendarPainter
不优先使用：复杂滑动动画、多选、列表联动、拉伸日历
```

## 19. 错误与诊断

错误展示分三层：

```text
第一层：主界面短提示
例如：天气更新失败，已显示缓存数据

第二层：设置页最近错误
显示时间、模块、简短原因

第三层：导出诊断日志
用户主动导出，自动隐藏 API Key
```

错误类型：

| 类型 | 示例 |
|---|---|
| NetworkError | 无网络、DNS 失败、TLS 失败 |
| ApiError | 401、403、429、接口限流 |
| ParseError | 返回字段缺失、JSON 解析失败 |
| LocationError | 城市搜索失败 |
| CalendarError | 农历或节气数据异常 |
| HolidayDataError | 假日 JSON 格式错误 |
| BootError | 开机自启失败 |
| LauncherError | 备用启动器打开失败 |
| SensorError | 室内传感器读取失败 |

## 20. 验收标准

### 20.1 主界面验收

1. 横屏模式下左侧显示大时钟，右侧显示月历。
2. 时间字号明显大于其他信息。
3. 当前日期在月历中有明显背景色、边框或反色。
4. 日期、星期、农历、天气、三日预报、Wi-Fi 状态同屏可见。
5. 不同主题下均可看清主信息。
6. 无网络时不出现大面积空白。

### 20.2 低版本验收

1. Android 4.1 设备或模拟器可安装。
2. Android 4.2 设备或模拟器可安装。
3. Android 4.4 设备或模拟器可安装。
4. Android 5.0+ 设备可安装。
5. 不使用 Android 5.0+ 才有的 UI API 作为必需路径。
6. 不依赖 Google Play Services。

### 20.3 日历验收

1. 月历显示完整。
2. 农历显示正确。
3. 节气显示正确。
4. 法定假日显示“休”。
5. 调休补班显示“班”。
6. 今天高亮显示。
7. 用户切换周起始后日历重新排布。

### 20.4 自启与 Launcher 验收

1. 开启开机自启后，设备重启能进入主界面。
2. 设置为默认 Home 后，按 Home 键回到本应用。
3. 能从本应用打开备用启动器。
4. 能从设置页关闭相关功能。
5. 不会造成用户无法返回系统桌面。

### 20.5 天气验收

1. 设置城市后能显示当前天气。
2. 能显示今天、明天、后天预报。
3. 天气刷新失败时显示缓存。
4. 错误提示不会遮挡主界面核心内容。
5. API Key 不出现在日志中。
